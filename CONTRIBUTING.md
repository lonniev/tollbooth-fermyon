# Contributing to tollbooth-fermyon

## Ground rules

- **`main` is protected.** All changes land through a pull request; no direct
  pushes. PRs require passing CI and an up-to-date, linear branch.
- **Commits must be signed and verified.** This repo requires cryptographically
  signed commits (GPG or SSH), verified by GitHub. Set it up once:

  ```bash
  # GPG
  git config user.signingkey <YOUR_KEY_ID>
  git config commit.gpgsign true
  # or SSH
  git config gpg.format ssh
  git config user.signingkey ~/.ssh/id_ed25519.pub
  git config commit.gpgsign true
  ```

  Then add the corresponding public key to your GitHub account under
  *Settings → SSH and GPG keys* so commits show **Verified**.

- **CI must be green.** Every PR runs: Rust (`cargo fmt`, `clippy -D warnings`,
  `cargo test`, `cargo component build`), Python (`ruff`, byte-compile), the
  bridge Worker (dry-run build), and a secret scan. Fix all of them before
  requesting review.

- **Never commit secrets.** Nostr `nsec` values, Neon connection strings,
  BTCPay keys, and API tokens stay in gitignored files (`.env`, `.env.local`,
  `*.local.json`) or platform secret stores — never in the tree.

## Component-specific dev

- `crypto/` — `cargo test` validates the primitives byte-for-byte against
  vectors generated by the `tollbooth-dpyc` wheel. Keep new primitives covered.
- `component/` — the Python WASI component; see `component/spike/FINDINGS.md`
  for the current build/run recipe.
- `bridge/` — the Cloudflare Worker; `npm run dev` for a local instance.

## The DRY rule

The [`tollbooth-dpyc`](https://github.com/lonniev/tollbooth-dpyc) wheel owns all
proof, ledger, pricing, vault, and crypto logic. This repo consumes it
**untouched** — adapt the runtime around the wheel; never fork or reimplement it.
